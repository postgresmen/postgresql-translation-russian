PostgreSQL 9.4 Increases Flexibility, Scalability and Performance
==================================================================

ОКТЯБРЬ 2014: Всемирная группа разработки PostgreSQL объявила о выходе PostgreSQL 9.4, новейшей версии лидирующей реляционной системы управления базами данных (СУБД) с открытыми исходными текстами. Этот релиз добавляет множество новых возможностей, повышающих настраиваемость, масштабируемость и производительность СУБД для различных потребностей пользователей, включая улучшенную поддержку JSON, репликацию и производительность индексов.
## OCTOBER 2014: The PostgreSQL Global Development Group announces the release of PostgreSQL 9.4, the latest version of the world's leading open source database system.  This release adds many new features which enhance PostgreSQL's flexibility, scalability and performance for many different types of database users, including improvements to JSON support, replication and index performance.

Гибкость
Flexibility
-----------

С новым типом данных JSONB пользователям больше не придется выбирать между реляционными и нереляционными хранилищами данных: PostgreSQL дает возможность воспользоваться преимуществами обоих. JSONB поддерживает быстрый поиск и выразительные поисковые запросы на базе обобщенных обратных индексов (GIN). Многочисленные вспомогательные функции получения и манипулирования данных JSON достигают равной или более высокой производительности по сравнению с популярными документно-ориентированными СУБД. JSONB позволяет объединить данные в таблицах БД и документах в единую среду хранения данных.
With the new JSONB data type for PostgreSQL, users no longer have to choose between relational and non-relational data stores: they can have both at the same time.  JSONB supports fast lookups and simple expression search queries using Generalized Inverted Indexes (GIN).  Multiple new support functions enable users to extract and manipulate JSON data, with a performance which matches or surpasses the most popular document databases.  With JSONB, table data can be easily integrated with document data for a fully integrated database environment.

"JSONB сближает PostgreSQL и сообщество разработчиков Javascript, позволяя хранить и запрашивать данные JSON с помощью встроенных средств СУБД. Node.js и другие инфраструктуры разработки приложений смогут работать с такой безопасной и отказоустойчивой СУБД как PostgreSQL, не жертвуя своей бессхемной моделью хранения данных", -— говорит Мэтт Солдо (Matt Soldo), менеджер по продукту в проекте Heroku Postgres.
"JSONB brings PostgreSQL to the JavaScript development community by allowing JSON data to be stored and queried natively. Node.js and other server-side JavaScript frameworks can benefit from the safety and robustness of PostgreSQL, while still storing data in the schema-less format that they prefer," said Matt Soldo, Product Manager, Heroku Postgres.

Масштабируемость
Scalability
-----------

Логическое декодирование, добавленное в версии 9.4, предоставляет новый интерфейс разработки приложений (API) для чтения, фильтрации и манипулирования потоковыми данными при репликации. Он лежит в основе средств репликации нового типа, таких как двунаправленная репликация (BDR), поддерживающая кластеры с несколькими ведущими узлами. Другие улучшения, такие как слоты репликации и реплики с задержкой по времени, облегчают управление серверами реплик и расширяют диапазон их использования.
In 9.4, Logical Decoding supplies a new API for reading, filtering and manipulating the PostgreSQL replication stream.  This interface is the foundation for new replication tools, such as Bi-Directional Replication, which supports the creation of multi-master PostgreSQL clusters.  Other improvements in the replication system, such as replication slots and time-delayed replicas, improve management and utility of replica servers.

"Появление логического декодирования стало основной причиной незамедлительного внедрения PostgreSQL 9.4 в производственные процессы“, -- говорит Марко Фавале (Marco Favale), менеджер картографической продукции в компании Navionics. "Возможность создания индивидуальных настраиваемых модулей расширений для вывода данных позволит нам напрямую собирать данные избранных таблиц и реплицировать изменения по нашему усмотрению, устраняя необходимость в более тяжеловесных и сложных в использовании решениях на основе триггеров".
"The main reason behind our immediate adoption of PostgreSQL 9.4 in production is the new Logical Decoding feature," said Marco Favale, Cartographic Production Manager of Navionics. "The ability to write custom and flexible output plugins will allow us to transparently collect changes on selected tables and replicate changes where we like, by removing heavier - and more complex to manage - trigger based
replication solutions."

"Zalando рассчитывает на стабильность и производительность сотен серверов PostgreSQL для непрерывного обслуживания запросов миллионов клиентов со всей Европы“, -— утверждает Валентин Гогичашвили (Valentine Gogichashvili), руководитель команды работы с базами данных компании Zalando Technologies. "Мы приветствуем появление встроенных реплик с задержкой по времени и начнем экспериментировать со средствами двунаправленной репликации сразу после их выхода".
"Zalando relies on the stability and performance of hundreds of PostgreSQL database servers to continuously serve millions of customers all around Europe," said Valentin Gogichashvili, Team Lead Database Operations at Zalando Technologies.  "We are excited to run time-delayed standby servers which work out of the box, and will evalutate the new bi-directional replication tools as soon as they are released."

Производительность
Performance
-----------

Версия 9.4 также содержит множественные улучшения, позволяющие добиться большей производительности каждого сервера PostgreSQL. Они включают в себя:
Version 9.4 also introduces multiple performance improvements which will allow users to get even more out of each PostgreSQL server.  These include:

* уменьшение размера индексов GIN на 50% и их ускорение в три раза;
* Improvements to GIN indexes, making them up to 50% smaller and up to 3X faster.
* материализованные представления с параллельными обновлениями для создания быстрых и своевременных отчетов;
* Concurrently updatable Materialized Views for faster, more up-to-date reporting.
* возможность ускоренной загрузки данных в кэш СУБД после перезагрузки, предоставляемая расширением pg_prewarm;
* Rapidly reload the database cache on restart using pg_prewarm.
* ускоренная параллельная запись в журнал транзакций PostgreSQL.
* Faster parallel writing to the PostgreSQL transaction log.

"Мы определенно воспользуемся параллельным обновлением материализованных
представлений, резервными серверами с задержкой по времени (повышающими
надежность процедур аварийного восстановления), также как и улучшениями
производительности, которые происходят обычно при выходе новой версии",
-- добавил Марко Фавале.
"We will definitely benefit from concurrent refresh of materialised
views, delayed standby servers (which will make our disaster recovery
even more robust) as well as the usual performance improvements every
new release carries on," added Marco Favale.

О проекте PostgreSQL
About PostgreSQL
----------------
О PostgreSQL: PostgreSQL является ведущей СУБД с открытыми исходными текстами, с глобальным сообществом из тысяч пользователей и разработчиков, объединяющим множество компаний и организаций. Проект PostgreSQL создается на основе более чем 25-летнего опыта проектирования и разработки, начавшейся в Калифорнийском университете Беркли, и в настоящее время разрабатывается беспрецедентными темпами. Продуманный набор возможностей PostgreSQL не только не уступает ведущим коммерческим СУБД, но и превосходит их развитой функциональностью, расширяемостью, безопасностью и стабильностью. Вы можете получить дополнительную информацию о PostgreSQL и присоединиться к нашему сообществу по адресу http://www.postgresql.org.
PostgreSQL is the leading open source database system, with a global community of thousands of users and contributors and dozens of companies and organizations.  The PostgreSQL Project builds on over 25 years of engineering, starting at the University of California, Berkeley, and has an unmatched pace of development today. PostgreSQL's mature feature set not only matches top proprietary database systems, but exceeds them in advanced database features, extensibility, security and stability.  Learn more about PostgreSQL and participate in our community at: http://www.postgresql.org.
